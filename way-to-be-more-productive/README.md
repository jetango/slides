* 这个梗来自 Malcolm Gladwell 的书 Outliers

* 还不带休息的
* 嫌多？降低点标准试试
* 1 * 365 * 3 = 1095 小时
* (5 + 10) * 52 * 3 = 2340 小时
* 业余爱好还好啦
* 但是如果成为专家，要有长期坚持的准备

## 好方法&好工具

* 目标与计划
    * 长期计划太遥远，记住就好不用想太多
    * 计划赶不上变化
    * 一般只做到周的计划
    * 专注于今天，现在
* 反馈与回顾
    * 记录做过的事情
    * 记录踩过的坑
    * 不怕忘
    * 没事可以拿出来笑自己以前傻

* 异步工作
    * 尽量少卡在别人的环节上
    * 自己掌握主动权
    * 不卡意味着并行少，需要关注的也少
    * 排除干扰，集中精力
    * 专注于现在

* 处理邮件
    * 把私人与工作分开两个邮箱
    * 邮件是工作规范，留记录
    * 其实更像工单性质
    * 定时检查，次数不宜多，一般我一天早晨看一次中午看一次
    * 因为紧急重要的事情是当面或电话沟通的
    * 把重点需要跟进的标记出来
    * 但不是现在处理
    * 可以考虑往Gmail备份一份，搜索比较好用

* 交换数据内容请使用纯文本
* 适当考虑简单的标记语言
* 字符编码请使用Unicode
* 源代码管理用 Git
* Git 托管
* 使用 Pull-Request
* IRC
    * 找国外开发者即时聊天一般只有这条路
    * irssi / XChat / Colloquy
* Slack - Be less busy.
    * 多人即使聊天，类IRC
    * 有归档
    * 能搜索！（赞）
    * 能放代码片段，附件，一样能搜索！（赞）
    * 基于Web，意味着跟系统无关
    * 有API可以自己扩展！（大赞）

## 请爱用键盘

* mouse-free
* 想哪打哪
* 快、准、稳
* 很多情况下只有键盘可以用
* Launcher应用
    * 不用找菜单，不用双击鼠标
    * Windows 7 以上自带
    * Launchy (Windows)
    * Spotlight (Mac OS X)
    * Alfred (Mac OS X) 好贵的，我没用过
    * Unix/Linux 比较注重这个，不论什么DE或WM一般都自带
* 深入一个编辑器
    * 好的编辑器提升效率
    * 扩展能力
    * Notepad++ (Windows)
    * Sublime Text 2/3 （$）
    * Atom (Mac OS X)
* 为什么推荐 Vim & Emacs
    * 30余年老字号
    * 老师傅都在用
    * 投入有回报
    * 几乎无所不能
* vim
    * 编辑器之神
    * Esc/HJKL
    * vim script
    * 扩展丰富
    * 脚本语言扩展（推荐用Python）

    * 神之编辑器
    * “Emacs是一个仅缺乏优秀编辑器的操作系统。”

## 挑选键盘
* 这不是广告，是经验
* 便宜好用，Dell SK8115，但是寿命短，已停产
* 机械键盘，入门推荐Cherry茶
* 茶轴、青轴适合打字，有段落感
* 红轴适中，也适合打字
* 黑轴，压力克数大，适合玩游戏
* 更小众的轴都是玩家玩的，就不推荐了
* 有无钢板
* 静电的坑太大，也不推荐了
* 104 / 87 / MINI 布局
* 自制、改造（DIY）
